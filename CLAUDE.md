# Neovim Configuration Notes

This is a plain Lua Neovim configuration. It was previously written in Fennel;
all Fennel sources and the compilation step have been removed — the `.lua`
files are the hand-written sources now.

## Key Information
- Configuration language: Lua (edit `.lua` files directly, no compile step)
- Entry point: `init.lua` (bootstraps lazy.nvim and the plugin list)
- Plugin configs live in `/plugin/` (auto-sourced by Neovim at startup):
  - `plugin/*.lua` - options, keymaps, colorscheme, misc
  - `plugin/configs/*.lua` - third-party plugin setup
  - `plugin/custom/*.lua` - homegrown mini-plugins
- Shared modules live in `/lua/user/` and are loaded with `require("user.<name>")`
- Exception: `list-element-text-objects.lua` files are generated by
  TypeScriptToLua - don't hand-edit those
- The `queries/fennel/` and fennel-related plugins (conjure, parinfer-rust,
  fennel.vim, nfnl) remain only for editing lisp files in other projects

## Levvy: custom fuzzy scorer for Telescope
- Telescope's `file_sorter`/`generic_sorter` are backed by **levvy**, a
  SIMD fuzzy-match scorer written in Zig (`~/code/tarshtein-distance`),
  when its shared library is installed
- Install/update the library with `~/code/tarshtein-distance/install.sh`
  (copies `liblevvy.so` into `<config>/zig/`, which is gitignored)
- Integration lives in `lua/user/levvy.lua` (FFI binding) and
  `lua/user/levvy-telescope.lua` (sorter); wired up in
  `plugin/configs/telescope.lua`
- When the library is missing, the config falls back to
  telescope-fzf-native automatically — nothing breaks on machines
  without the build

## Notes System Integration

### Journal Setup
- **Location**: `~/notes/.private/journal/YYYY/YYYY-MM-DD.norg`
- **Format**: Neorg files (`.norg` extension)
- **Organization**: Year-based subdirectories for better organization
- **Keybinding**: `<leader>nj` - Open today's journal entry

### Notes Keybindings (`<leader>n` namespace)
- `<leader>nj` - Open today's journal entry
- `<leader>ns` - Save notes (runs `~/notes/.bin/save.sh` with streaming notifications)
- `<leader>np` - Pull notes from git (with streaming notifications)
- `<leader>nn` - Navigate to notes directory (opens Oil file explorer)

### Explore Keybindings
- `<leader>en` - Explore notes (opens in new tab)
- `<leader>ev` - Explore vim config (opens in new tab)

### Search Keybindings
- `<leader>sv` - Search vim config files
- `<leader>zv` - FZF search vim config files

### Streaming Notifications
- Uses `mini.notify` for floating notification windows (top-right corner)
- Background script execution with `vim.fn.jobstart`
- Real-time stdout/stderr streaming via `notes_notify` wrapper
- Uses `vim.schedule()` to avoid "fast event context" errors

## Text Wrapping Configuration
- **File types**: markdown, norg, org, text
- **Settings**:
  - `textwidth = 80` - Auto-wrap at 80 characters
  - `formatoptions += "t"` - Enable auto-wrapping while typing
  - `<leader>tw` - Toggle the formatoptions `l` flag (reformat existing long lines or not)

### Manual Text Wrapping
- `gqq` - Format current line
- `gqip` - Format current paragraph
- `gq}` - Format until next paragraph
- Visual mode + `gq` - Format selected text

## Neorg Plugin
- **Plugin**: `nvim-neorg/neorg`
- **Configuration**:
  - Lazy-loaded on `.norg` filetype
  - Workspace: `~/notes` directory
  - Modules: `core.defaults`, `core.concealer`, `core.dirman`
  - Icon preset: `basic`

## File Explorer Function Enhancement
- **Function**: `open_in_explorer(dir, opts)` in `plugin/keymaps.lua`
- **Optional parameter**: `{ new_tab = true }` - Opens explorer in new tab

## LSP Configuration Notes
- **Vue server**: Use `vue_ls` (not `volar` or `vue_language_server`)
- **Path expansion**: Always use `vim.fn.expand("~/path")` for tilde expansion in `vim.system`
- **Deprecated functions**: `vim.tbl_islist` is deprecated, use `vim.islist` instead

## Reloading
- There is no build step; restart Neovim to pick up config changes
  (`<leader>qr` quits all so you can restart)

## Common Patterns

### Keybinding with Function
```lua
vim.keymap.set({ "n", "v" }, "<leader>key", function()
  -- do the thing
end, { desc = "Description" })
```

### Background Command with Notifications
```lua
vim.fn.jobstart({ "command", "args" }, {
  cwd = vim.fn.expand("~/directory"),
  on_stdout = function(_, data, _)
    for _, line in ipairs(data) do
      if line and line ~= "" then
        vim.schedule(function() notes_notify(line, vim.log.levels.INFO) end)
      end
    end
  end,
  on_exit = function(_, code, _)
    vim.schedule(function()
      notes_notify("Command completed!", vim.log.levels.INFO)
    end)
  end,
})
```
